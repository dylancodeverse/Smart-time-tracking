Principe de Séparation des Responsabilités (SRP - Single Responsibility Principle):


1️⃣ VehiculeDB (Data Source)

    Accède directement à la base de données (SQLite).
    Effectue les opérations CRUD (Create, Read, Update, Delete).
    Ne contient aucune logique métier, juste des requêtes SQL.


2️⃣ VehiculeRepository (Abstraction du Data Layer)

    Permet de découpler l’accès aux données en centralisant la logique d’accès.
    Facilite le changement de source de données (ex. passer de SQLite à une API REST sans impacter la logique métier).
    Sert d'intermédiaire entre la base de données et le ViewModel.
